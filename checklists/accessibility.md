# Accessibility Checklist

Use this checklist for public-facing teaching artifacts, especially HTML
modules, wizards, games, and dashboards.

## Minimum Checks

- Text contrast is readable on the approved palette, including buttons, cards,
  disabled states, and chart labels.
- Keyboard users can reach every interactive control in a sensible order.
- Buttons and links have visible focus states.
- Headings follow a logical order and do not skip levels for visual styling.
- Images, diagrams, and icons have useful alt text or are marked decorative when
  they add no information.
- Color is not the only way to understand a result, warning, or status.
- Motion is limited and respects reduced-motion preferences when animations are
  used.
- Forms and reflection fields have labels, helper text, length limits, and clear
  error messages.
- Mobile layout is usable without horizontal scrolling.

## Verification Note

Record the checks performed in `VERIFICATION.md`. If a visual, chart, or game
mechanic cannot be made fully accessible in the first pass, state the limitation
plainly and provide a fallback explanation or instructor note.
